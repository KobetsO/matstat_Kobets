def factorial(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result

print(factorial(5))
def fac(n):
    if n == 0:
        return 1
    return fac(n - 1) * n

n = int(input("Введіть число: "))
print(fac(n))

Алгоритм обчислення факторіалу за допомогою циклу for має лінійну складність, оскільки цикл виконується n разів, де n — вхідне число. Кожна ітерація циклу займає постійний час O(1), оскільки це проста операція множення. Отже, загальна асимптотична складність алгоритму — O(n).

  1. Оцінити асимптотичну складність рекурсивного алгоритму обчислення факторіалу.

    • Кількість викликів функції: Кількість рекурсивних викликів прямо пропорційна значенню n, оскільки ми зменшуємо n на 1 на кожному кроці, аж до досягнення n=0.
    • Час виконання кожного виклику: Кожен виклик виконує одну операцію (множення) і займає постійний час O(1).
    Отже, загальна кількість викликів — n, а кожен виклик займає час O(1).
def fibonacci(n):
    if n == 0:
        return 0
    if n in (1, 2):
        return 1
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(30))

  1. Оцінити асимптотичну складність рекурсивного алгоритму обчислення 𝑛-го числа Фібоначчі.

    • Кількість рекурсивних викликів: Кількість викликів зростає експоненціально з кожним зменшенням n. Кожен рекурсивний виклик для n призводить до двох додаткових викликів: одного для n−1 і іншого для n−2.
    • Часова складність: Кількість рекурсивних викликів можна оцінити як O(2n), оскільки для кожного рівня ми маємо приблизно подвоєння кількості викликів. Це відбувається через дублювання обчислень, наприклад, при обчисленні F(n−1) і F(n−2), обчислення для менших значень часто повторюються.
    Асимптотична складність Загальна асимптотична складність рекурсивного алгоритму обчислення n-го числа Фібоначчі — O(2 n). Це експоненціальна складність, оскільки кількість операцій подвоюється з кожним збільшенням n.
def merge(left, right):
    result = []
    i, j = 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result += left[i:]
    result += right[j:]
    return result

def mergesort(lst):
    if len(lst) < 2:
        return lst
    middle = len(lst) // 2
    left = mergesort(lst[:middle])
    right = mergesort(lst[middle:])
    return merge(left, right)


a = [6, 5, 3, 1, 8, 7, 2, 4]
print(mergesort(a))

  1. Оцінити асимптотичну складність алгоритму сортування злиттям, використовуючи головну теорему рекурсії.

Рекурентне співвідношення для сортування злиттям У процесі роботи алгоритму сортування злиттям масив розбивається на дві частини, кожна з яких сортується окремо, а потім об’єднується. Це дає нам таке рекурентне співвідношення для часу виконання T(n), де n — розмір масиву: T(n)=2T(n/2)+O(n).

Застосування головної теореми рекурсії

Головна теорема рекурсії застосовується для рекурентних співвідношень виду: T(n)=aT(n/b)+O(nd)

Контрольні питання

1. Надати визначення складності задачі із символом Ω.

  • Ω-нотація використовується для позначення нижньої межі часової складності алгоритму. Це означає, що алгоритм виконується щонайменше за Ω(f(N)) операцій, де f(N) — функція від розміру вхідних даних N. Формально, якщо існує додатне число C і таке значення N₀, що для всіх N ≥ N₀ виконується f(N) ≤ C * g(N), тоді можна сказати, що f(N) = Ω(g(N)).

2. Функція часової складності має вигляд: 𝐹(𝑁)=𝑁3+7𝑁2−14𝑁. Як записати асимптотичну складність у 𝑂-нотації?

  • Для знаходження асимптотичної складності в O-нотації необхідно залишити лише домінуючий член, тобто той, який зростає найшвидше при збільшенні N. У цьому випадку це N3, тому: F(N)=O(N3)

3. Функція часової складності має вигляд: 𝐹(𝑁)=1.01𝑁+𝑁10. Як записати асимптотичну складність у 𝑂-нотації?

  • Тут домінуючий член — N10, оскільки він зростає швидше за 1.01N. Тому: F(N)=O(N10)

4. Функція часової складності має вигляд: 𝐹(𝑁)=𝑁1.3+10log2𝑁. Як записати асимптотичну складність у 𝑂-нотації?

  • Домінуючий член тут — N1.3, так як він зростає швидше за 10 log2 𝑁. Тому: F(N)=O(N1.3)

5. У чому полягає ідея розпаралелювання обчислень і для чого вона використовується? Які з алгоритмів, наведених у цій лабораторній роботі, дозволяють можливість розпаралелювання?

  • Ідея розпаралелювання полягає в тому, щоб розбити великі обчислення на менші підзадачі, які можуть виконуватися одночасно на кількох процесорах або ядрах. Це підвищує швидкість обчислень і ефективність використання ресурсів. Розпаралелювання застосовується для великих задач, де обробка може бути розділена, як, наприклад, у задачах обробки даних або в алгоритмах обчислювальної геометрії. Можливість розпаралелювання залежить від конкретного алгоритму — наприклад, алгоритми, які не мають сильних залежностей між своїми частинами, підходять для розпаралелювання.

6. Які існують способи підвищення обчислювальної швидкості алгоритмів? Який з них є найефективнішим?

  • Способи підвищення швидкості включають оптимізацію алгоритму (вибір більш ефективного алгоритму), розпаралелювання обчислень, використання більш швидкого обладнання (швидших процесорів або графічних карт), а також оптимізацію коду (усунення зайвих операцій). Найефективніший спосіб залежить від конкретної задачі, але зазвичай вибір більш ефективного алгоритму має найзначніший вплив на швидкість.
