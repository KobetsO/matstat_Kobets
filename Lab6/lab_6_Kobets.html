1. Написати функцію pop_n(), що видаляє елементи стека з його початку до номера n включно

class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        return None

    def is_empty(self):
        return len(self.items) == 0

    def size(self):
        return len(self.items)


def pop_n(stack, n):
    for _ in range(min(n, stack.size())):
        stack.pop()

2. Оцінити асимптотичну складність (у середньому і в найгіршому разі) процедур search, insert і delete роботи зі стеком

  1. Вставка (insert):
    • Опис: Додавання елементу у вершину стеку.
    • Складність: O(1) у середньому і в найгіршому випадку.
    • Вставка елемента у стек завжди займає однаковий час, оскільки додавання відбувається у вершину.
  2. Видалення (delete):
    • Опис: Видалення елементу з вершини стеку.
    • Складність: O(1) у середньому і в найгіршому випадку.
    • Видалення елемента з вершини стеку завжди займає однаковий час.
  3. Пошук (search):
    • Опис: Пошук елементу у стеку.
    • Складність: O(n) у середньому і в найгіршому випадку.
    • У найгіршому випадку потрібно переглянути всі елементи стеку, щоб знайти потрібний елемент або переконатися, що його немає у стеку.
Контрольні питання

1. Що таке стек і які операції можна виконувати зі стеком?

Стек — це структура даних типу LIFO (Last In, First Out), де елементи додаються та видаляються з вершини. Операції, які можна виконувати зі стеком:

  • push(x): додати елемент x у вершину стеку.
  • pop(): видалити і повернути елемент з вершини стеку.
  • peek() або top(): повернути елемент з вершини стеку, не видаляючи його.
  • is_empty(): перевірити, чи порожній стек.
  • size(): повернути кількість елементів у стеку.

2. Яка основна відмінність між стеком та чергою?

  • Стек: працює за принципом LIFO (останній прийшов, перший пішов). Елементи додаються і видаляються з одного кінця (вершини).
  • Черга: працює за принципом FIFO (перший прийшов, перший пішов). Елементи додаються в кінець і видаляються з початку.

3. Як ви можете реалізувати стек за допомогою масиву і за допомогою зв’язаного списку? Які переваги та недоліки кожного підходу?

Реалізація стеку за допомогою масиву

Переваги:

  • Простота реалізації.
  • Швидкий доступ до елементів за індексом.

Недоліки:

  • Фіксований розмір (якщо використовувати статичний масив).
  • Необхідність збільшення розміру (якщо використовувати динамічний масив), що може призвести до витрат часу на копіювання. 
class ArrayStack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        return None

    def peek(self):
        if not self.is_empty():
            return self.items[-1]
        return None

    def is_empty(self):
        return len(self.items) == 0

    def size(self):
        return len(self.items)

Реалізація стеку за допомогою зв’язаного списку

Переваги:

  • Гнучкий розмір (можна додавати або видаляти елементи без необхідності збільшення розміру).
  • Використання пам’яті тільки для необхідних елементів.

Недоліки:

  • Більш складна реалізація.
  • Більш повільний доступ до елементів порівняно з масивом через необхідність проходити список. 
class Node:
    def __init__(self, data=None):
        self.data = data
        self.next = None

class LinkedListStack:
    def __init__(self):
        self.head = None
        self._size = 0

    def push(self, item):
        new_node = Node(item)
        new_node.next = self.head
        self.head = new_node
        self._size += 1

    def pop(self):
        if self.is_empty():
            return None
        popped_item = self.head.data
        self.head = self.head.next
        self._size -= 1
        return popped_item

    def peek(self):
        if self.is_empty():
            return None
        return self.head.data

    def is_empty(self):
        return self.head is None

    def size(self):
        return self._size

4. Які є застосування стека та черги в програмуванні і реальному житті?

Застосування стека

У програмуванні:
  • Рекурсія: Використання стеку для зберігання станів рекурсивних викликів.
  • Обробка виразів: Застосування стеку для перевірки балансу дужок та обчислення значень математичних виразів (наприклад, в оберненій польській нотації).
  • Скасування дій: Використання стеку для зберігання історії дій у текстових редакторах, щоб можна було скасувати останні зміни.
У реальному житті:
  • Стопка тарілок: При додаванні або видаленні тарілок з вершини стопки використовується принцип LIFO.
  • Кнопка “Назад” в браузері: Зберігає історію відвіданих сторінок у стеку, щоб можна було повернутися назад.

Застосування черги

У програмуванні:
  • Обробка задач: Використання черги для зберігання задач, які мають бути оброблені в порядку надходження (наприклад, в операційних системах для планування процесів).
  • Затори даних: Використання черги для управління потоками даних, щоб уникнути перевантаження системи.
У реальному житті:
  • Черга в магазині: Люди стоять у черзі і обслуговуються в порядку надходження.
  • Обробка заявок: Використання черги для обробки заявок, наприклад, у банках або службах підтримки клієнтів.
